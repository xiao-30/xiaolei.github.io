关于生命周期的一些知识点

John Doe 发布于 2024-07-25

最后更新于 2024-07-31

生命周期有哪些？ （vue2中）

  • beforeCreate : 组件实例被创建之前
  • created : 组件实例已经完全创建
  • beforeMount : 组件挂载之前
  • mounted : 组件挂载到实例之后
  • beforeUpdate : 组件数据发生变化，更新之前
  • updated : 组件数据更新之后
  • beforeDestory : 组件数据销毁之前
  • destoryed : 组件数据销毁之后

一旦进入组件会执行哪些生命周期？

  • 一旦进入组件 就会执行前四个生命周期
    1. beforeCreate
    2. created (在前两个生命周期中是没有dom的)
    3. beforeMount
    4. beforeMount

父组件引入子组件，生命周期执行的先后顺序是什么？

  • 父组件的 beforeCreate
  • 父组件的 created
  • 父组件的 beforeMount
  • 子组件的 beforeCreate
  • 子组件的 created
  • 子组件的 beforeMount
  • 子组件的 mounted
  • 父组件的 mounted

如果引入多个子组件，那么也先执行 父组件的前三个生命周期，然后再执行子组件的前四个生命周期，当引入多个子组件的时候，按照子组件引入到父组件的先后顺序依次执行，最后执行父组件的 mounted

在 created 中如何获取 dom?

当生命周期中有异步请求的时候，不论在 beforeCreate 中还是 created,beforeMount 中，都可以获取dom
只要写异步代码，获取dom 是在异步请求中获取的，因为异步永远慢于同步同步。
例如使用 setTimeout/ 请求/ Promise.xxxx()
还可以使用vue系统 内置的 this.$nextTick() 方法 都可以获取dom

为什么不在 beforeCreate 中发送请求？

如果请求是封装在 methods 中的，在 beforeCreate 中调用请求的方法时，beforeCreate是拿不到 methods 里面的方法的（会报错）

beforeCreate 和 created 的区别？

beforeCreate 没有 $data 也没有 $el
created 有 $data， 没有 $el
beforeCreate 中是不能拿到 methods 中的方法的
created 中是可以拿到 methods 中的方法的

发送请求在 created 中还是在mounted 中？

这个问题要看项目和业务的情况，因为组件加载的顺序是 先执行父组件的前三个生命周期（beforeCreate , created , beforeMount） ，然后再执行子组件的前四个生命周期（beforeCreate , created , beforeMount , mounted），当我们的业务需要先加载子组件的数据，然后再加载父组件的数据，那么就放在 mounted 中，如果父组件的数据不需要加载，那么就放在 created 中，所以，发送请求在created 中，还是在 mounted 中 ，要看我们的业务情况和需求

加入 keep-alive 会执行那么生命周期

keep-ailve 是vue 中的内置组件，能在组件切换的过程中将状态保存在内容中，防止重复渲染DOM
keep-ailve 包裹动态组件时，会缓存不活动的组件实例，而不是销毁他们
keep-ailve 缓存当前组件

  • 如果使用keep-ailve 包裹动态组件，当前组件会额外增加两个生命周期 activated 和 deactivated

  • 如果当前组件加入了 keep-ailve， 第一次执行会执行五个生命周期

  1. beforeCreate
  2. Created
  3. beforeMount
  4. mounted
  5. activated

  • 如果当前组件加入了 keep-ailve， 第二次或者第 N 次执行会执行一个生命周期

    1. activated

  • 如果当前组件没有加入 keep-ailve， 第二次或者 第 N 次执行会执行四个生命周期

    1. beforeCreate
    2. Created
    3. beforeMount
    4. mounted

  • 你在什么情况下使用过生命周期？说一说生命周期使用场景

    1. created 当有父组件和子组件的时候，根据项目需求需要先加载父组件然后再加载子组件的时候。父组件在发送请求的时候，需要在 created 中发送请求，因为同时拥有子组件和父组件的时候，父组件会先执行前三个生命周期（beforeCreare,created , beforeMount）,然后再去执行子组件的前四个生命周期 （beforeCreate,created , beforeMount , mounted）,所以父组件发送请求的时候应该在 created 中发送请求。
    2. mounted 当有父组件和子组件的时候，根据项目需求需要先加载子组件然后再加载父组件的时候。父组件在发送请求的时候，需要在 mounted 中发送请求，因为同时拥有子组件和父组件的时候，父组件会先执行前三个生命周期（beforeCreare,created , beforeMount）,然后再去执行子组件的前四个生命周期 （beforeCreate,created , beforeMount , mounted）,然后再去执行父组件中的 mounted ,所以当子组件需要先加载的时候，父组件后加载的时候，父组件的请求应该在 mounted 中发送请求。
    3. created,mounted都可以 当是单个组件的时候，发送请求时，没有特殊要求的话，在前四个生命周期中，拿一个生命周期中都可以。
    4. activated 当我们需要不断的切换或者点击组件的时候，需要缓存组件，防止重复渲染DOM，所以，在组件切换的过程中，会执行 activated 和 deactivated
    5. destoryed 关闭页面记录视频的时间，初始化的时候从上一次的历史记录中获取视频播放的时间
